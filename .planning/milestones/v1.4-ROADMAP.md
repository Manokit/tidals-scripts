## Overview

Fix withdraw functionality by replacing broken bank.search() detection with sprite-based findAndVerifyItem() across all withdrawal methods.

## Phases

### Phase 12: Fix Withdraw

**Goal**: Fix finding items and actually withdrawing them from the bank
**Depends on**: v1.3 Humanization complete
**Plans**: 1 plan

Plans:
- [x] 12-01: Replace bank.search() with sprite-based findAndVerifyItem()

**Details:**

The core issue was that OSMB's `bank.search(Set.of(itemId))` was failing to detect items that were visually present in the bank. The fix replaced all three uses of this broken method with the already-working `findAndVerifyItem()` which uses visual sprite matching + menu verification:

1. `findItemByScrolling()` - scroll search detection
2. `searchAndWithdraw()` - primary item detection and withdrawal
3. `searchAndFillInventory()` - fill inventory detection and withdrawal

Added `getWithdrawActions(int amount)` helper to standardize withdraw action string generation.
