# Milestone v1.0: Fairy Ring Mode

**Status:** SHIPPED 2026-01-16
**Phases:** 1-3
**Total Plans:** 4

## Overview

Add fairy ring support to TidalsSecondaryCollector for players without Drakan's Medallion. The script auto-detects equipment to choose between Ver Sinhaza mode (existing) and Fairy Ring mode (new), then handles collection, banking, and return accordingly.

## Phases

### Phase 1: Mode Detection

**Goal**: Auto-detect fairy ring vs Ver Sinhaza mode from equipment
**Depends on**: Nothing (first phase)
**Plans**: 1 plan

Plans:
- [x] 01-01: Equipment detection and mode selection logic

**Details:**
- Added Mode enum with VER_SINHAZA and FAIRY_RING values
- Implemented mode detection in verifyRequirements()
- Detection: Dramen staff + inventory bloom + ardy cloak = FAIRY_RING
- Detection: Drakan's medallion + equipped bloom = VER_SINHAZA
- Added isFairyRingMode() helper for future phases

### Phase 2: Collection

**Goal**: Collect fungus at 3-log tile with inventory bloom
**Depends on**: Phase 1
**Plans**: 1 plan

Plans:
- [x] 02-01: 3-log tile collection and inventory bloom casting

**Details:**
- Added THREE_LOG_TILE constant at (3474, 3419, 0)
- Added THREE_LOG_POSITIONS for log detection
- Implemented castBloomFromInventory() method
- Made collect(), determineState(), detectFungusPositions() mode-aware

### Phase 3: Banking & Return

**Goal**: Bank via Zanaris, return via monastery fairy ring
**Depends on**: Phase 2
**Plans**: 2 plans

Plans:
- [x] 03-01: Zanaris banking via fairy ring
- [x] 03-02: Monastery return and BKR validation

**Details:**
- Added MORT_MYRE_FAIRY_RING position at (3469, 3431, 0)
- Added ZANARIS_BANK_PATH (8 waypoints to bank chest)
- Implemented useZanarisBanking() for fairy ring teleport
- Added MONASTERY_TO_FAIRY_PATH (10 waypoints)
- Implemented useFairyRingReturn() with BKR validation
- Script terminates safely if BKR not configured
