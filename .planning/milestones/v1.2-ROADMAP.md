# Milestone v1.2: Verified Withdrawals

**Status:** SHIPPED 2026-01-14
**Phases:** 7-9
**Total Plans:** 3

## Overview

Make searchAndWithdrawByName robust by verifying items exist before withdrawing. Added sprite-based scroll position detection, item ID search with tapGetResponse verification, and refactored withdrawal flow to eliminate blind fixed-offset tapping.

## Phases

### Phase 7: Scroll Position Detection

**Goal**: Sprite-based at-top/at-bottom detection for bank scroll position
**Depends on**: Phase 6
**Plans**: 1 plan

Plans:
- [x] 07-01: Add isAtTop() and isAtBottom() methods

**Details:**
- Added SCROLLBAR_TOP_Y (334) and SCROLLBAR_BOTTOM_Y (507) constants
- Used sprite 789 for top detection, sprite 791 for bottom detection
- Fixed Y coordinate detection pattern for reliable scroll position checking

### Phase 8: Item ID Screen Search

**Goal**: Search visible bank area for item by ID with tapGetResponse verification
**Depends on**: Phase 7
**Plans**: 1 plan

- Added searchBankForItem combining scroll + verify for full bank search
- Case-insensitive contains matching for item names

### Phase 9: Robust Withdrawal Flow

**Goal**: Refactor searchAndWithdrawByName with new verified item search flow
**Depends on**: Phase 8
**Plans**: 1 plan

Plans:

## Milestone Summary

**Key Decisions:**
- Used different sprite halves for scroll position (789 top, 791 bottom)
- Fixed Y coordinate detection at 334 (top) and 507 (bottom)
- Made findItemInVisibleBank private - internal helper only
- First slot verification only since search consolidates results
- isAtBottom() over canScrollDown() for scroll termination

**Issues Resolved:**
- searchAndWithdrawByName no longer uses blind fixed-offset tapping
- Items are now verified via tapGetResponse before withdrawal action
