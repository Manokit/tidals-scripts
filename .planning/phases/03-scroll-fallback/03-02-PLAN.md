  <name>Task 1: Add end-of-scroll detection to BankScrollUtils</name>
  <files>utilities/src/main/java/utilities/BankScrollUtils.java</files>
  <action>
    Add scroll state detection methods to BankScrollUtils:

    1. scrollDownAndCheck(Script script) method:
       - Get current bank item slots BEFORE scroll (use bank.search(Set.of()) for all items)
       - Store the slot IDs/positions visible
       - Call scrollDown()
       - Wait for scroll animation (300-500ms pollFramesHuman)
       - Get bank item slots AFTER scroll
       - Compare before/after - if same slots visible, we hit the bottom
       - Return true if bank contents changed (more to scroll), false if at end

    2. scrollUpAndCheck(Script script) method:
       - Same pattern as scrollDownAndCheck but going up
       - Return true if contents changed, false if at top

    3. scrollToTop(Script script) method:
       - Loop scrollUpAndCheck() until it returns false (at top)
       - Add max iterations safeguard (20 scrolls max)
       - Return true if successfully scrolled to top

    Key insight from research: OSRS bank at end-of-scroll shows same items after scroll attempt.
    Compare visible slot positions, not item IDs (items don't move, view does).
  </action>
  <verify>Build compiles: cd tidals-scripts && JAVA_HOME=$(/usr/libexec/java_home -v 17) gradle :utilities:compileJava</verify>
  <done>scrollDownAndCheck(), scrollUpAndCheck(), scrollToTop() methods exist and compile</done>
</task>

<task type="auto">
  <name>Task 2: Integrate scroll fallback into searchAndWithdraw</name>
  <files>utilities/src/main/java/utilities/BankSearchUtils.java</files>
  <action>
    Modify searchAndWithdraw() to use scroll fallback when search doesn't find item:

    1. Add overload: searchAndWithdraw(Script, itemId, amount, keepSearchOpen, useScrollFallback)
       - Default useScrollFallback=true in existing overloads

    2. When item not found after search AND useScrollFallback=true:
       - Clear current search first
       - Log "item not found via search, trying scroll fallback"
       - Call BankScrollUtils.scrollToTop(script) to start from top
       - Loop: search for item, if not found call scrollDownAndCheck()
       - Continue until item found OR scrollDownAndCheck returns false (end of bank)
       - Max 30 scroll iterations safeguard

    3. If found via scroll:
       - Log success with scroll count
       - Proceed with withdrawal

    4. If not found after full scroll:
       - Log "item not found in bank after full scroll"
       - Return false

    Important: Search-first is the fast path. Only fall back to scrolling when search fails.
    The search feature in OSRS is very reliable, so scroll fallback is for edge cases.
  </action>
  <verify>Build compiles: cd tidals-scripts && JAVA_HOME=$(/usr/libexec/java_home -v 17) gradle :utilities:compileJava</verify>
  <done>searchAndWithdraw() uses scroll fallback when search fails to find item</done>
