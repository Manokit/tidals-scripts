Add a withdrawBatch() method to BankSearchUtils that:

1. Takes parameters: Script script, List<WithdrawalRequest> requests
2. Returns: BatchWithdrawalResult
3. Pre-condition check: bank must be visible, return empty result with all items failed if not
4. Process each request in order:
   - Call searchAndWithdraw(script, request.getItemId(), request.getAmount(), true) for all but last item (keepSearchOpen=true)
   - Call searchAndWithdraw(script, request.getItemId(), request.getAmount(), false) for last item (clears search)
   - Track success/failure in result object
5. Log summary: "batch withdrawal complete: X/Y items succeeded"
6. Return the result

Add a convenience overload:
- withdrawBatch(Script script, WithdrawalRequest... requests) that converts varargs to List

Design notes:
- Continue processing even when items fail (partial success is valid)
- Use keepSearchOpen=true for efficiency (avoids reopening search for each item)
- Last item always clears search to leave bank in clean state

Build the jar after implementation.
